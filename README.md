# m-igame-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。所收录的页面均以静态 HTML 文件形式存在，不依赖特定域名或后端服务，适合长期保存与直接浏览。

## 项目简介

本仓库旨在提供一个轻量、开放的内容存档空间。所有页面均为独立的 HTML 文件，可随时通过浏览器直接打开查看。仓库内容不针对任何具体网站或平台，仅作为个人/团队项目页面的集合与备份。

## 目录结构

```
m-igame-journal/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/             # 页面共用静态资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：主要归档目录，按需存放各类独立页面。
- `assets/`：存放多个页面可能用到的样式、脚本或图片资源，避免重复引用。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖外部服务或数据库。
- 页面之间相互独立，无跨域或链接依赖。
- 文件名建议使用英文小写加连字符（如 `my-page.html`），便于识别与访问。
- 若页面需要引用资源，请使用相对路径（相对于仓库根目录或 `assets/` 目录）。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在必要时更新 `index.html` 中的导航链接。
- 更新页面：直接修改对应 HTML 文件即可，无需额外配置。
- 删除页面：移除文件并清理相关引用（如有）。
- 资源管理：共用资源请放入 `assets/`，避免在每个页面中重复存放。

## 注意事项

- 本仓库不包含任何动态内容或服务器端逻辑。
- 页面内容应遵循 GitHub 社区准则，避免包含违规或敏感信息。
- 由于是静态归档，建议定期检查链接与资源路径的有效性。
